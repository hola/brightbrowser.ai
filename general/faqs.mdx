  <Accordion title="Can I force BrightBrowser.AI to use a specific country?">
    <Text>BrightBrowser.AI <strong>automatically</strong> optimizes IP routing for the best performance. However, if you need to scrape from a specific country, you can manually set the location.</Text>

    <Text><strong>How?</strong><br />
    Just append <code>-country-XX</code> to your <strong>authentication string</strong>, where <code>XX</code> is the <strong>ISO 3166-1 alpha-2 country code</strong>.</Text>

    <Text><strong>Example: Targeting the US in Puppeteer</strong></Text>

    <CodeGroup title="Puppeteer">
      ```js
      const BBAI_WS_ENDPOINT = `wss://${USERNAME}-country-us:PASSWORD@brd.superproxy.io:9222`;
      ```
    </CodeGroup>

    <Text><strong>Want all of Europe?</strong><br />
    Use <code>-country-eu</code> to dynamically select an EU-based IP.</Text>

    <Tip>
      <strong>Heads-up:</strong> For <strong>bot-heavy</strong> websites, manually selecting a country may actually decrease success rates. Let BrightBrowser.AI handle it automatically when possible!
    </Tip>
  </Accordion>

  <Text variant="h2">üîé Debugging & Troubleshooting</Text>

  <Accordion title="How do I debug my BrightBrowser.AI session?">
    <Text>Seeing <strong>what your browser is doing in real time</strong> is critical for debugging. BrightBrowser.AI integrates seamlessly with <strong>Chrome DevTools</strong> for full visibility.</Text>

    <Text><strong>Two ways to debug:</strong></Text>

    <ul>
      <li><strong>Control Panel:</strong> Open the debugger in Bright Data‚Äôs dashboard.</li>
      <li><strong>Remote Code:</strong> Attach DevTools from your script.</li>
    </ul>

    <Text><strong>Example: Open DevTools from your Puppeteer script</strong></Text>

    <CodeGroup title="Puppeteer">
      ```js
      const page = await browser.newPage();
      const client = await page.target().createCDPSession();
      await client.send('Page.inspect', { frameId: page.mainFrame()._id });
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="How do I access the BrightBrowser.AI Debugger?">
    <Tabs>
      <Tab title="Through the Control Panel">
        <Text>Open <strong>Bright Data Dashboard</strong> ‚Üí <strong>My Zones</strong> ‚Üí Select <strong>BrightBrowser.AI</strong> ‚Üí Click <strong>"Chrome DevTools Debugger"</strong>.</Text>
      </Tab>

      <Tab title="Via Code (Remote)">
        <Text>Use the <strong>CDP API</strong> to open a <strong>DevTools debugging session</strong> from your script:</Text>

        <CodeGroup title="Puppeteer">
          ```js
          const page = await browser.newPage();
          const client = await page.target().createCDPSession();
          const { url: inspectUrl } = await client.send('Page.inspect', { frameId: page.mainFrame()._id });
          console.log(`Inspect your session here: ${inspectUrl}`);
          ```
        </CodeGroup>
      </Tab>
    </Tabs>
  </Accordion>

  <Text variant="h2">‚ö° Performance & Best Practices</Text>

  <Accordion title="Why does my page take longer to load than expected?">
    <Text>Some websites have <strong>complex anti-bot measures</strong>, which require extra processing to <strong>bypass protections</strong>.</Text>

    <Text><strong>To ensure smooth navigation, increase the timeout threshold:</strong></Text>

    <CodeGroup title="Puppeteer">
      ```js
      await page.goto('https://example.com', { timeout: 120000 });
      ```
    </CodeGroup>

    <Tip>
      Always set a <strong>higher timeout (2+ minutes)</strong> for bot-protected websites.
    </Tip>
  </Accordion>

  <Accordion title="How long can my session stay active?">
    <ul>
      <li><strong>Idle sessions</strong> (no interaction) <strong>auto-close after 5 minutes</strong>.</li>
      <li><strong>Max session duration:</strong> 30 minutes. After that, your session will time out.</li>
    </ul>
  </Accordion>


    <Text><strong>If you get JSON in response:</strong> ‚úÖ You're successfully connected!</Text>
